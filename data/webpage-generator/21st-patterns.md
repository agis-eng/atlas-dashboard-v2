# 21st.dev Pattern Notes

Source role: pattern intelligence for modern UI interactions and section treatments. Do not copy components verbatim; adapt the underlying idea to the page concept.

## High-signal patterns observed
- **AI prompt box / prompt input:** large input surface, strong affordance, supporting action chips, premium focus states
- **AI chat shells:** layered conversation panels, anchored composer, motion that emphasizes response flow
- **Agent plan / workflow cards:** staged steps, progress framing, plan/result contrast, visible action hierarchy
- **Animated input surfaces:** micro-motion around input focus, hover, send states, and active affordances
- **Dense media cards:** preview-first cards that combine thumbnail/video, title, author, and engagement metadata cleanly

## When to use these patterns
- Use **prompt-box patterns** for AI tools, search-driven workflows, and “tell us what you need” entry points
- Use **chat-shell patterns** when the interface is conversational and continuity matters
- Use **plan/workflow cards** when the product turns intent into steps, status, or automation
- Use **animated inputs** sparingly to make the main action feel premium
- Use **dense media cards** when showcasing examples, templates, or component libraries

## Why they work
- The primary action is visually obvious
- Motion is attached to user intent, not random decoration
- Surfaces feel layered and interactive instead of flat
- Preview content and metadata are balanced well
- Cards communicate state, not just style

## Adaptation rules
- Preserve clarity over spectacle
- Keep one dominant interactive surface per screen
- Use motion to reinforce focus and flow
- Avoid stacking too many “fancy” interaction motifs on one page
- Translate the pattern to the business context; do not leave it looking like an AI template gallery

## Good uses in Atlas-generated pages
- high-value lead capture hero with a premium prompt-style intake box
- AI/automation service pages with plan/result workflow sections
- dashboard pages with stronger task/status hierarchy
- case-study or example galleries with richer preview cards
